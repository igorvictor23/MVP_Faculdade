Lista de Requisitos do MVP

Requisitos Funcionais (REF):

ID Requisito Descrição
REF-001 Mostrar informações A plataforma deve mostrar aos usuários informações referentes as opções de ecoturismo da cidade (ex: horários, localização, eventos, etc.).
REF-002 Login de administradores administradores devem poder fazer login para acessar áreas restritas da aplicação.
REF-003 Área dos administradores A aplicação deve possuir uma área para administradores realizarem modificações nos conteúdos do site.

Requisitos Não Funcionais (RENF):

Requisitos Não Funcionais de Desempenho:

ID Requisito Descrição
RENF-001 Tempo de resposta A aplicação deve responder a requisições dos usuários em até 2 segundos.
RENF-002 Capacidade de usuários simultâneos A aplicação deve suportar um grande número de usuários simultâneos sem degradação de desempenho.

Requisitos Não Funcionais de Segurança:

ID Requisito Descrição
RENF-003 Autenticação de usuários A aplicação deve garantir que apenas usuários autenticados possam acessar áreas restritas.

Requisitos Não Funcionais de Usabilidade:

ID Requisito Descrição
RENF-004 Interface amigável A aplicação deve possuir uma interface amigável e intuitiva para os usuários.

Requisitos Não Funcionais de Portabilidade:

ID Requisito Descrição
RENF-005 Compatibilidade com navegadores A aplicação deve ser compatível com os principais navegadores (Chrome, Firefox, Safari, etc.).
RENF-006 Compatibilidade com dispositivos móveis A aplicação deve ser responsiva e funcionar corretamente em dispositivos móveis.
